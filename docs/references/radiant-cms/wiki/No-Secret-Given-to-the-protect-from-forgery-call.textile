h2. Symptoms

When upgrading from pre-0.6.8, if you get this error on the login screen:

<pre><code>No :secret given to the #protect_from_forgery call.  Set that or use a session store capable of generating its own keys (Cookie Session Store).</code></pre>

h2. The Solution

If you haven't run @rake production db:migrate@, you'll have to do so first. Open @environment.rb@ and change:

<pre><code>config.action_controller.session_store = :active_record_store</code></pre>

to

<pre><code>config.action_controller.session_store = :cookie_store</code></pre>

When you get a @CGI::Session::CookieStore::TamperedWithCookie@ error afterwards, simply clear your cookies and reload the page.
